# 《晨读刮刮乐》代码说明文档

本文档面向需要参与维护、联调或二次开发的协作人员，说明仓库内 **目录结构、状态与数据流、各模块职责及扩展入口**。阅读后应能独立理解「从麦克风到开奖庆祝」的完整逻辑，并知道在何处修改行为。

## 1. 技术栈与运行方式

| 类别 | 选型 |
|------|------|
| 框架 | Vue 3（`<script setup>` + Composition API） |
| 语言 | TypeScript（`strict`） |
| 构建 | Vite 6 |
| 样式 | Tailwind CSS v4（`@import "tailwindcss"` + `@theme` 变量，见 `src/style.css`） |
| 状态 | Pinia |
| 持久化 | `@vueuse/core` 的 `useStorage`（同步 `localStorage`） |
| 特效 | `canvas-confetti`（庆祝弹窗中的 Fireworks 式彩带） |

**常用命令（在项目根目录）：**

- `npm install`：安装依赖  
- `npm run dev`：本地开发（需 HTTPS 或 localhost 才能稳定使用麦克风，依浏览器策略而定）  
- `npm run build`：`vue-tsc` 类型检查 + 生产构建  
- `npm run preview`：预览构建产物  

**路径别名：** `@/` 指向 `src/`（配置于 `vite.config.ts` 与 `tsconfig.app.json`）。

## 3. 核心业务：有限状态机（FSM）

运行时状态字段为 `GameState.status`，取值如下（定义于 `src/types/index.ts`）：

| 状态 | 含义 | 典型触发 |
|------|------|----------|
| `idle` | 待机，可点击「开始晨读」 | 首屏、`resetGame()`、从庆祝弹窗「再读一遍」 |
| `recording` | 收音中，进度可增长 | `onStartRound()` 成功拿到麦克风后 |
| `finished` | 进度已满，正在展示开奖动效（老虎机落定或刮刮乐自动刮） | `useGameEngine` 将进度推到 100% |
| `celebrating` | 奖励高光 + 全屏彩带 | `enterCelebrating()`（由子组件事件触发） |

**与 PRD 的对应关系简述：**

- **进度规则：** 仅当 `currentDb >= config.targetDb` 时，在 `recording` 状态下按「经过的实时时间 / 目标时长」增加 `progress`（见 `useGameEngine`）。  
- **中奖结果：** 进度到达 100% 时调用 `calculateWinner()`，按权重从 `config.rewards` 中抽取 `winningReward`（在展示开奖前已确定）。  
- **配置持久化：** `GameConfig` 整包存入 `localStorage` 键名 `chen-du-config`；`GameState` 不持久化，刷新页面会丢失当前轮进度。

## 4. 数据模型（`src/types/index.ts`）

### 4.1 `RewardItem`

- `id`：字符串唯一标识（新增项建议使用 `crypto.randomUUID()`）  
- `name`：奖励文案  
- `weight`：整数权重；中奖概率 ≈ `weight / 总权重`（实现见 `pickWeightedReward`）

### 4.2 `GameConfig`（持久化）

- `mode`：`'slot'` 刮图片（老虎机） / `'scratch'` 刮刮乐  
- `targetDb`：达标音量阈值，范围语义上与 UI 一致为 **0–120 的相对刻度**（非物理分贝绝对值）  
- `targetDurationSeconds`：达标状态下「读满进度」所需秒数（控制中心按 5 秒对齐）  
- `rewards`：奖池数组  

### 4.3 `GameState`（运行时）

- `currentDb`：当前音量展示值（与 `useAudioAnalyzer` 同步，便于控制中心展示）  
- `progress`：0–100  
- `status`：上节 FSM  
- `winningReward`：本轮结果，可为 `null`（例如奖池为空或权重为 0）

## 5. 全局状态：`useGameStore`（`src/stores/useGameStore.ts`）

### 5.1 导出状态

- **`config`**：`useStorage<GameConfig>('chen-du-config', 默认值)`，读写即触发本地持久化。  
- **`runtimeState`**：`ref<GameState>`，仅内存，页面刷新重置。

### 5.2 导出方法

| 方法 | 作用 |
|------|------|
| `calculateWinner()` | 按权重随机写入 `runtimeState.winningReward` |
| `resetGame()` | 进度归零、`status → idle`、清空中奖项与展示用 `currentDb` |
| `enterCelebrating()` | `status → celebrating`，用于打开庆祝弹窗与彩带 |

### 5.3 权重抽奖算法

`pickWeightedReward`：在 `(0, totalWeight]` 上取随机数，线性扫描累减权重，为常见「轮盘赌」实现。奖池为空或总权重 ≤ 0 时返回 `null`。

## 6. 音频与进度：`composables`

### 6.1 `useAudioAnalyzer.ts`

**职责：** 在用户手势后请求麦克风，用 `AnalyserNode` 读取时域数据，计算 RMS 并映射为 **0–120 的整数** `currentDb`（便于与滑块阈值同屏对比）。

**要点：**

- `start()`：先 `stop()` 再 `getUserMedia` → `AudioContext` → `createMediaStreamSource` → `createAnalyser`，`fftSize = 2048`，用 `requestAnimationFrame` 持续更新 `currentDb`。  
- `stop()`：`cancelAnimationFrame`、停止所有音轨、`audioContext.close()`，避免占麦与内存泄漏。  
- `onUnmounted` 时自动 `stop()`。

**注意：** 浏览器通常要求 **用户点击等手势** 后才能顺利启动音频上下文；失败时由 `App.vue` 捕获并展示 `errorMsg`。

### 6.2 `useGameEngine.ts`

**职责：** 每一帧（`requestAnimationFrame`）在「正在录音且 `status === 'recording'`」时，若 `currentDb >= targetDb`，则按 `deltaSec / targetDurationSeconds * 100` 增加 `progress`。

**满进度时：**

1. 调用 `calculateWinner()`  
2. `progress = 100`，`status = 'finished'`  

此后由 UI 分支决定何时 `enterCelebrating()`（见下一节）。

**设计意图：** 用 rAF 的 `delta` 做计时，避免 `setInterval` 与渲染不同步导致的漂移。

### 6.3 `useFireworksConfetti.ts`

**职责：** 封装 [canvas-confetti](https://github.com/catdad/canvas-confetti) 文档中的 **Fireworks** 思路：左右两侧交替 `confetti()`，在固定总时长内用 `requestAnimationFrame` 循环调用。

**参数：** `runFireworksConfetti(durationMs)`；庆祝弹窗使用 **5000ms**。选项中带 `disableForReducedMotion: true`，尊重系统「减少动效」偏好。

## 7. 根组件编排：`App.vue`

### 7.1 组合关系

- `storeToRefs(useGameStore())` 解构出响应式 `config`、`runtimeState`。  
- `useAudioAnalyzer()` 提供 `currentDb`、`startMic`、`stopMic`。  
- `isRecording = computed(() => runtimeState.status === 'recording')`，传入 `useGameEngine(currentDb, isRecording)`（只读计算属性；引擎内不得对其赋值，满进度时仅通过 `status → finished` 使其自动变为 `false`）。  
- `watch(currentDb)`：把分析结果写回 `runtimeState.currentDb`（供控制中心「当前音量」展示）。  
- `watch(status)`：凡 `status !== 'recording'` 则 `stopMic()`，避免非收音阶段仍占麦。

### 7.2 用户操作

| 操作 | 行为 |
|------|------|
| 开始晨读 | `startMic()` 成功后 `progress=0`、`status=recording`、清空上轮 `winningReward` |
| 控制中心 | `panelOpen = true`，抽屉内修改 `config` 通过 `@update:model-value` 写回 `config`（持久化） |
| 刮刮乐揭开 | 子组件 `@revealed` → `enterCelebrating()` |
| 老虎机落定 | 子组件 `@settled` → `enterCelebrating()` |
| 再读一遍 | `resetGame()` + `scratchSession++`（强制重挂刮刮乐子树，重置画布） |
| 庆祝弹窗·进入控制中心 | `status → idle` 并打开抽屉 |

### 7.3 模式切换与 `scratchSession`

- `config.mode === 'slot'` 时渲染 `RewardSlotMode`，否则 `RewardScratchMode`。  
- 刮刮乐组件使用 `:key="scratchSession"`：**每轮「再读一遍」递增 key**，确保 `StratchCard` 重新挂载，遮罩与画布状态干净。

## 8. UI 组件说明

### 8.1 `layout/MainContainer.vue`

页面外壳：渐变背景、`@theme` 糖果色变量、标题与默认副文案；主内容通过默认插槽插入。

### 8.2 `game/ProgressBar.vue`

**入参：** `progress`（0–100）、`isLoudEnough`（当前是否 ≥ 阈值）。  

**表现：** 宽度用内联 `width`（非 transform）绑定百分比；未达标时灰度 + `animate-pulse`；达标时可选流光层（scoped 动画）。

### 8.3 `game/RewardSlotMode.vue`（刮图片 / 老虎机）

**逻辑概要：**

- `recording && loudEnough`：短间隔 `setInterval` 随机切换展示的奖励名（盲盒感）。  
- `finished`：停表，展示 `winningReward.name`（或奖池为空提示），CSS 弹跳动效后 `emit('settled')`。  
- `celebrating` 时父组件仍将 `finished` 视为真，保持最终文案展示。

### 8.4 `game/RewardScratchMode.vue`（刮刮乐）

- 内嵌 `StratchCard`，默认槽位展示 `winningReward.name`。  
- 监听 `finished === true`：`nextTick` 后调用 `cardRef.autoReveal(2000)`，实现 **无需手指** 的自动刮开。  
- `@scratch-all` → `emit('revealed')`。

**注意：** Canvas 遮罩色须为 **实色**（`fillStyle` 兼容），不要使用 CSS 渐变字符串作为 `maskColor`。

### 8.5 `components/StratchCard.vue`

**Props（与对外文档一致）：** `scratchPercent`、`imageUrl`、`maskColor`、`fillStyle`、`font`、`text`、`radius`、`scratchRadius` 等。

**事件：**

- `scratchStart` / `scratchEnd`：开始/结束刮擦（自动刮开时也会触发）  
- `scratchAll`：达到阈值或自动流程结束并揭开

**实例方法（`defineExpose`）：**

- `reset()`：收起遮罩状态、重算尺寸并重绘遮罩。  
- `autoReveal(durationMs?)`：沿 **Z 字形路径** 用 `destination-out` 擦除；若结束时透明像素比例仍不足 `scratchPercent`，则 **整画布擦除** 保证能完成；最后 `maskHidden = true` 并 `emit('scratchAll')`。

**实现要点：** 鼠标/触摸坐标按 `getBoundingClientRect` 与 `canvas` 实际宽高比换算；事件监听使用 `AbortController` 便于卸载与重置。

### 8.6 `celebration/ConfettiModal.vue`

- `Teleport to="body"`，高 `z-index`。  
- `open === true` 时 `watch` 触发 `runFireworksConfetti(5000)`。  
- 展示 `rewardName`；按钮「再读一遍」「进入控制中心」对外 emit。

### 8.7 `control-panel/SettingsDrawer.vue`

- 侧滑抽屉；`open` 与 `update:open` 支持 `v-model:open`。  
- `modelValue` 为完整 `GameConfig`，子项变更通过 `patch(partial)` 合并后 `emit('update:modelValue')`。  
- 内嵌：`VolumeSlider`（阈值 + 实时音量柱）、时长秒数输入、`RewardListEditor`。

### 8.8 `control-panel/VolumeSlider.vue` / `RewardListEditor.vue`

- **VolumeSlider：** `modelValue` 为阈值；`liveDb` 为当前音量，用于右侧迷你柱状条。  
- **RewardListEditor：** 列表编辑 `rewards`，新增行使用 `crypto.randomUUID()`；整表以 `update:rewards` 回传。

## 10. 协作开发：常见改动入口

| 需求 | 建议修改位置 |
|------|----------------|
| 调整达标难度 / 默认时长 | `useGameStore` 默认值或控制中心 UI；进度公式在 `useGameEngine` |
| 改抽奖算法 | `useGameStore` 中 `pickWeightedReward` |
| 改老虎机速度 / 落定等待 | `RewardSlotMode` 中 `setInterval` 间隔与 `settled` 延时 |
| 改自动刮开速度 / 路径 | `StratchCard` 的 `autoReveal`、`buildZPath` |
| 改彩带样式 / 时长 | `useFireworksConfetti.ts` 与 `ConfettiModal` 中的调用参数 |
| 持久化键名 / 默认奖池 | `chen-du-config`、`constants/defaultRewards.ts` |
| 接入真实物理分贝或其它传感器 | 主要改 `measureDisplayDb` 的映射关系，并统一与 `targetDb` 语义 |

## 11. 已知注意点与调试建议

1. **麦克风权限：** 若 `getUserMedia` 拒绝，`App.vue` 会显示中文错误文案；需在浏览器设置中允许站点使用麦克风。  
2. **状态与麦克风：** `status !== 'recording'` 时会 `stopMic()`；从 `finished` 到 `celebrating` 不再录音，符合「读完即停」的课堂场景。  
3. **类型与构建：** 提交前执行 `npm run build`，确保 `vue-tsc` 无报错。  
4. **`canvas-confetti` 类型：** 项目内 `src/types/canvas-confetti.d.ts` 提供最简声明；升级库时可对照官方类型再扩充。
